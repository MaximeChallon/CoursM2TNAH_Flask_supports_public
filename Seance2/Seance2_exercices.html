REV-1: Les décorateurs

Voici un bout de code, soit une fonction grosse_multiplication décorée par chronometre.

@chronometre
def grosse_multiplication(multiplicateur):
    multiplicande = 1000000000000000*9999
    resultat = 0
    for i in range(1,multiplicateur):
        resultat += (multiplicande * multiplicateur *i)
    return  resultat

Développer le décorateur chronometre de manière à ce qu'un appel print(grosse_multiplication(56897*2)) affiche:

  • le temps d'exécution du calcul
  • le résultat du calcul

Pistes:

  • le calcul du temps pris revient à soustraire l'heure de fin à l'heure de début de traitement
  • un package Python permet d'accéder aux dates et aux heures

Exemple de sortie:

 La fonction grosse_multiplication a pris (en secondes) 0.02800130844116211 7366833379465092126000000000000000

APP-1 : requests et les routes à paramètres

  • Ecrire une route GET Flask nommée retrieve_wikidata
  • Cette route acceptera un paramètre qui sera l'identifiant Wikidata
  • Grâce au package requests (et à sa documentation), appeler Wikidata et récupérer la réponse correspondant à cet identifiant
  • Retourner à notre route retrieve_wikidata ce qui a été reçu

Le résultat attendu dans le navigateur est un clone de la page Wikidata, sans le CSS et le JS.

APP-3 : lire un fichier CSV, traiter les données et les restituer

A partir du jeu de données des examens de permis de conduire en 2012, créer une application qui permette de visualiser les données par lot de 50 à partir d'une route Flask sous la forme d'un tableau. Les données restituées attendues sont les suivantes:

  • Code postal
  • Ville
  • Adresse de l'auto-école
  • Nom de l'auto-école
  • Candidats au permis B
  • Taux de réussite au permis B

Le jeu de données est encodé en ISO-8859-1.

La route prendra deux paramètres afin d'effectuer la pagination dans l'url:

  • offset
  • from

Plusieurs librairies/méthodes sont possibles pour traiter les données brutes et les utiliser:

  • créer une boucle sur la lecture des lignes depuis la librairie csv
  • utiliser la librairie pandas

Il est conseillé d'utiliser pandas pour une meilleure efficacité. La correction proposée utilisera pandas.

APP-4 : un peu de Javascript

A partir de l'exemple Seance2/statics vu en cours, intégrer du JavaScript de différentes manières:

  • dans le template conteneur, intégrer une fonction JS qui sera commune à toutes les pages; appeler cette fonction dans la page pays

REF-1 : maquette HTML du devoir

A partir de la maquette HTML et du code HTML générés en début de semestre dans le cours sur HTML et CSS:

  • tenter de le découper en templates, conteneurs et sous-éléments

  • construire les appels aux ressources CSS, JS et autres

Aucune correction ne sera proposée pour cet exercice puisqu'il est propre à chaque groupe. C'est en revanche un bon moyen de commencer la transcription du code HTML vers l'application Flask.
