Objectifs principaux du cours

  • utiliser une bibliothèque Python en s’appuyant sur sa documentation
  • organiser la conception et la réalisation d’une application web avec back- et front-office
  • découvrir un ORM (Object-Relational Mapping)

Description pratique du cours

Ce cours est construit comme un pas à pas dans le développement d’une application Web Python reposant sur une base de données. Nous prendrons comme exemple des données du World Factbook de la CIA. Plusieurs frameworks Python existent pour le développement Web (Django, Flask): Flask sera utilisé.

Documents fournis à chaque cours et déposés sur le Moodle de la classe:

  • la présentation
  • un ensemble de fichiers HTML (déployés ici) comprenant :
    • des explications détaillées sur ce qui a été évoqué lors du cours
    • les exercices
    • les ressources et les exemples du cours
    • des liens vers le code source des applications développées
  • un dépôt Github avec le code source de l’application est accessible à tous: lien. Si Git n'est pas familier, un zip avec le code source pourra être fourni.

Il est normal de ne pas tout comprendre et maîtriser dès le cours. L’enjeu est de reprendre ce qui a été vu, de refaire ce qui a été fait en cours. Je suis disponible par mail et répondrai au plus vite en cas d'interrogations.

Les exercices

A la fin de chaque cours, des exercices seront proposés pour le cours suivant. Ils seront parfois nombreux, mais ils restent optionnels dans leur totalité. Il est possible de les effectuer en groupe, les échanges sont toujours plus utiles que de longues heures de questionnements devant son écran.

  • soit un simple énoncé sera donné afin de développer l’application demandée (une fois la base d'une application Flask vue en cours, je vous conseille de garder une copie de cette structure de base quelque part afin d'éviter de devoir tout redévelopper à chaque fois)
  • soit une application initialisée est fournie, et il faudra y développer des fonctionnalités précises

Aucune correction ne sera effectuée en cours, les exercices sont à m’envoyer par mail (de préférence quelques jours avant le cours) pour que je puisse corriger et apporter des pistes de correction/améliorations. Une note de participation basée sur la réalisation (i.e. tentative de résolution) d'au moins un exercice à l'issue de chaque cours sera attribuée en fin d'UE.

3 types d’exercices seront proposés :

  • révisions sur ce qui a été vu dans le cours précédent (REV)
  • approfondissement de notions vues en cours, ou découverte de librairies ou de nouvelles méthodes de développement (APP)
  • réflexions sur une problématique autour d'un exercice afin de préparer le cours suivant (REF)

Evaluation

Par groupe de 2 ou 3 (groupes différents de ceux de l'UE3), les étudiants réaliseront le développement d’une application web correspondant à l’un des 3 sujets ci-dessous. Ce travail est à rendre pour le 31 mars 2023.

Note: des exemples de rendus de devoirs des années précédentes sont disponibles dans ce dépôt. Ils étaient auparavant réalisés individuellement.

Sujets

  • Réaliser une application lisant, modifiant et écrivant dans une base de données relationnelle (SGBDR au choix: SQLITE, PostreSQL, MariaDB) préremplie (possibilité d’utiliser les données utilisées dans l’UE3)
    • plusieurs pages de rendus de données seront demandées, par exemple: un catalogue, un endpoint JSON, des cartes, des graphes, etc.
    • la page de catalogue et le endpoint JSON devront intégrer des filtres sur les données. Faire de même pour les cartes et les graphes serait un plus.
    • notions appelées: tables HTML, SQL, une bibliothèque JS pour les cartes et/ou les graphes
  • Réaliser une application d’annotation d’images utilisant le standard IIIF : une application de prises de notes sur un document des collections de la BNF, avec affichage des images à la demande.
    • Pour chaque image affichée, possibilité de rédiger des commentaires et d’avoir des champs de description (numéro de page, titre, couleur, etc).
    • En complément, optionnel : pouvoir éditer pour chaque ouvrage un rapport pdf (ou autre format) reprenant l’image, les champs de description et les commentaires
    • notions appelées : formulaires Flask-WTF, SQL, API, IIIF
  • Réaliser une application qui cherche dans Wikidata des monuments ou des lieux (par ex. les châteaux en France, les musées, les bibliothèques…), puis dans BANO et/ou OpenStreetMap (OSM) les coordonnées de localisation et si possible les polygones: afficher le tout (données Wikidata, liens) dans une carte dynamique
    • Pas d’écriture de données dans une base demandée pour ce sujet, la lecture depuis des API suffit
    • une variante pourrait être de créer un formulaire qui permette d’affiner les requêtes Wikidata: sur une région de notre choix, ou avoir le choix entre châteaux/musées/bibliothèques/etc.
    • notions appelées: package “requests” Python, API REST, web de données, JS Leaflet (ou autre framework)

Ces trois types de sujets peuvent être réalisés avec d'autres données. Les étudiants devront penser l'application en amont et se répartir les fonctionnalités et les recherches au sein de chaque groupe. La mise en place de cette organisation projet sera notée dans le cadre de l’UE4.
